Data structures and algorithms

Data : Data is represented by the values held temporarily in the program's data area or recorded permanently on a file. The different data values may be related to each other; which needs to be organised to form a proper data structure. The program follows certain rules to access, manipulate and process the structured data.

Data Structure : A data structure refers to an organised way of arrangement of data, related to each other in the memory which allows the program to handle the data in an efficient way. A data structure can thus be represented as follows :

Data structure = data organization + allowed operations.
For example consider a single dimensional array in C declared as follows :
int a[5];
In the above declaration the array structure 'a' holds 5 elements of integer type. The index sunscripting starts from 0 and ends with 4. The array elements are accessed by their index positions.

types of data structures

A data structure is a logical method of representing data in memory using the simple and complex data tyes provided by the language.
The data structure can be classified into following two types:

simple data structure
These data structures are generally built from fundamental data types i.e, int, float, char, etc. Following data structures can be termed as simple data structures :
  • Array
  • Structure

compound data structure
These data structures are formed by using simple data structures and are more complex. Its two types are :
  • Linear data structures
  • Non-linear structures

Linear data structures. These are single level data structures, having their elements in a sequence. Examples of such structures are :
  • Stack
  • Queue
  • Linked
    list

Non-linear data structures. These are multilevel data structures. Examples of non-linear data structures are :
  • Tree
  • Graph

Array

It is a collection of homogenous (similar type) data elements. An array is also called linear data structure. It's elements are stored in computer memory in a linear fashion.

What is C++ ?

C++ is an object oriented programming language. It is a superset of C, that is, C with extentions to support OOP. It was developed by Bjarne Stroustrup of AT&T Bell Labs in 1986 as a means of making complex programs easier to write and maintain. The name C++ is a combination of C and the increment operator '++' indicating an incremental development of C.

C++ is an object oriented programming language. It was developed by Bjarne Stroustrup in 1983 at the AT&T Bell Laboratories, New Jersey, USA. It is basically a super set of C. which provided low level features and Simula 67, which provided the class concept. The version 1.0 became available commercially in 1985, version 2 in 1989 and version 3 in 1992.

    It has the following characteristics:
  1. Reduces complexity while solving problems.
  2. Correctness of results is ensured.
  3. Affordable in terms of hardware and other resources.
  4. Easier and cheaper for integrating existing software facilities and libraries.
  5. Portable i.e, can be used on different types of computers with little or no change in the programs.
C++, was originally named as 'C with classes. The name is derived from the 'increment' operator in C, which is '++. So C++ is an incremented version of C. C++ can be used for writing procedural code like C but its real strength lies in the writing of object-oriented programs.

An object oriented program is a collection of discrete objects, which are self contained collections of both data structures and functions that interact with other objects.

C++ adds classes, inheritance, function overloading and operator overloading. With the help of these one can create abstract data types, inherit properties from existing data types and use polymorphism. Thus we can say that C++ is an advancement of C. providing an additional set of object-oriented facilities.

Characteristics of object oriented languages

The fundamental idea behind Objectoriented programming languages is to combine into a single unit both data and the functions that operate on the data. Such a unit is called an 'object'. To create such a unit or object a prototype is defined. The prototype defination is the class. A class therefore is a major component of C++.

Class

Class describes the data and its behaviour or functionality. Objects are said to be 'members' of classes. This is the very much like what you see in nature, where Animal is a class and Cat is an object of this class. The basic features of animals, that is - 2 eyes, 2 ears, legs, etc... are common to all the Animals. Thus Cat which is an object of class Animal inherits these characteristics like colour, their nature, etc...
Just as int a predefined data-type in C++ can have many variables declared of its type, e.g, int a, count, total amount; declares 4 variables, all of the integer type. Similarly, you can define many objects of the same class. A class is a data-type defined by the user describing the data it represents and the function to be used to manipulate the data. It serves as a plan / template. Defining the class does not create any object. The class only describes how the object must be created and how it will behave.
User defined data type: A class is also a userdefined data type that determines the data structures and operations associated with that type.

Implementation of Object Oriented Programming Concepts in C++

"C with classes" was the original name given by the originator, Stroustrup initially. which now-a-days is popularly known as C++. Classes and objects are the most important features of C++. The class implements OOP features and ties them together. We have already seen that a structure groups different type of elements and a function organizes the program actions to perform the desired task.

DEFINITION OF A CLASS

A class in C++ combines related data and functions together. It makes a data type which is used for creating objects of this type. Classes represent real world entities that have both data type properties (characteristics) and associated operations (behaviour). The syntax of a class definition is shown below: 

class name_of_class
{
    private : variable declaration; //Data Member
              function declaration; //Member Function (method)
    protected : variable declaration;
                function declaration;
    public : variable declaration;
             function declaration;
};

Object

An object is created from a class. Each object is an instance of that class and has the data and behaviour descibed in that class. The term object and instance of a class can be used interchangably.
  • Physical Objects. e.g. Electrical components in a circuit design problem, Elements of the computer - user environment: Windows, Menus, Mouse, Keyboard, Graphics - objects (line, circle, etc.)
  • Programming Constructs Stacks, Customized Arrays. Linked lists, Binary trees.
  • User-defined data types: Angles, Complex numbers, Points on the plane.
  • Collections of data: Dictionary, an inventory.
  • Components in Computer games: Positions in a board game (chess, checkers), Opponents and friends in adventure games.

Encapsulation and Data Hiding

Encapsulation and Data hiding. A class defintion is a combination of data and its methods, an encapsulation. Encapsulation enables 'Data hiding' where the data can be declared to be private to the object and therefore its manipluation is possible only through a function of that class. This feature with all OO languages also makes it possible for the developer to change the data structure of an object without affecting the operation of the program.

Inheritance

Classes are divided into subclasses. Like the Animal, described above, can be divided into subclasses like Land animal, Water Animal etc. Animal in turn may be a subclass of the Fauna class. Thus Land Animal inherits all or some properties of the higher classes and can add properties of its own. The principle here is that each subclass shares common characteristics with the class from which it has been derived. In addition to the characteristics shared with other members of the class, each subclass has its own particular characteristics. In C++, the original class that is, the main class is called the 'Base class'. Other classes can be defined that share the 'Base class' characteristics and add their own as well. These are called 'Derived Classes. The derived class can be a base class of another class.

Reusability

The concept of inheritance provides an important extension to the idea of reusability Once a class has been written, created and debugged, it can be distributed to other programmers for use in their own programs. This is called reusability. A programmer can take an existing class, and without modifying it, add additional features and capabilities to it. This is done by deriving a new class from the existing one.

Polymorphism and Overloading

Polymorphism means "many forms and allows the programmer to create multiple definitions for operators and functions. During run-time depending on the context a particular definition of the operator or function will be used.

Operator overloading:

An existing operator like + used with integer values (built-in data type) constitutes addition. When a new data type is defined, the same operator can be made to operate in a particular manner on the new data type. This capability of an operator to operate on many data types or in a new manner. constitutes "operator overloading". Overloading is a kind of polymorphism. For eg is normally used to add numbers. This operator can be overloaded to add strings (concatenation) or sets (union) and so on. Similarly functions also can be overloaded as shown in the next section. The terms 'Overloading' and 'Polymorphism are used interchangeably.

Function overloading:

An overloaded function appears to perform different activities depending on the kind of data that is sent to it. A function can be overloaded depending on the factors like the number of arguments, the kind of arguments etc. The compiler can also distinguish between overloaded functions with the same number of arguments, provided their data types are different.
